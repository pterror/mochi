## hologram: $tick refinements

after explaining $tick to hazel, pterror raised two good points:

**per-channel intervals** — `$tick` currently ignores `$if` conditions when reading
the interval, so you can't do `$if channel.id === "xxx": $tick 5m`. noted as a
future improvement; would need initTickTimers to be async and evaluate facts with
channel context.

**simultaneous ticking** — the original implementation set up one timer per
channel×entity, so an entity bound to 3 channels would tick in all 3 at once.
fixed: now one timer per entity, round-robins across channels. prisma shows up in
one place at a time.

**`{{duration(idle_ms)}}` in facts** — pterror pointed out `formatDuration` already
exists and macros already evaluate ExprContext. so hazel can write
`it has been {{duration(idle_ms)}} since the last message` directly in prisma's
facts. no new code needed.
